Overview

Introduction to Bluetooth Low Energy (BLE)

Bluetooth Low Energy (BLE) is a wireless communication standard designed for short-range data exchange with very low power consumption. It is used in devices like fitness trackers, heart rate monitors, smart home sensors, medical devices, and more.

Unlike classic Bluetooth, BLE does not maintain a continuous connection. Devices advertise themselves periodically, and a central device (such as your laptop or phone) can scan, connect, and interact with them on demand.

📡

Central

The device that initiates the connection. In this lab, your browser is the central.

🔌

Peripheral

The BLE device being connected to (sensor, gadget). It advertises itself and hosts a GATT server.

📈

GATT

Generic Attribute Profile — the protocol that organises BLE data into Services and Characteristics.

📋

UUID

Unique identifier for a Service or Characteristic. Standard ones are 16-bit (e.g. 0x180F); custom ones are 128-bit.

ⓘ
Browser requirement
The Web Bluetooth API works only in Google Chrome and Microsoft Edge on desktop or Android. It does not work in Firefox or Safari.
Step 1

Scan for Nearby BLE Devices

Goal: Find a nearby BLE peripheral and select it.
  1. Make sure the BLE device you want to connect to is powered on and within range (≈10 m).
  2. Open the app at index.html in Chrome or Edge.
  3. Click the 🔍 Scan for BLE Devices button.
  4. A native browser dialog will appear listing nearby BLE devices. Select your device and click Pair.
  5. The selected device name and ID will appear in the Selected Device panel below the button.

What happens in the code

// The browser opens its built-in BLE device picker.
// The user selects one device.
const device = await navigator.bluetooth.requestDevice({
  acceptAllDevices: true
});
⚠
Security note: requestDevice() can only be called in response to a user gesture (button click). The browser will not let a page scan for BLE devices automatically on load.
Step 2

Connect to the GATT Server

Goal: Open a BLE connection and access the device's data structure.

After selecting a device, you must explicitly connect to it. BLE distinguishes between a device being known and being connected. Connecting establishes the ACL (Asynchronous Connection-Less) data channel and opens the remote GATT server.

  1. Click the 🔗 Connect to Device button (enabled after selecting a device).
  2. Wait 1–3 seconds while the BLE handshake completes.
  3. The badge in the top-right corner will change from DISCONNECTED to CONNECTED.
  4. The log will print: ✅ Connected to GATT server!

What happens in the code

// device.gatt is the BluetoothRemoteGATTServer object.
// .connect() opens the physical BLE link.
const server = await device.gatt.connect();
Step 3

Discover Primary Services

Goal: List all top-level services offered by the device.

A Service is a logical group of related data. Each device can expose multiple services. For example:

UUID Standard Service Name
0x180F Battery Service
0x180D Heart Rate Service
0x181A Environmental Sensing
0x1800 Generic Access
128-bit UUID Vendor-specific / custom service
  1. Click 📢 Discover Services.
  2. The list of services will appear below the button, each showing its UUID.
  3. Click on a service to select it (it will highlight in blue).
  4. The log will show all discovered UUIDs.

What happens in the code

// Returns an array of BluetoothRemoteGATTService objects.
const services = await server.getPrimaryServices();
Step 4

Discover Characteristics

Goal: List all data points inside the selected service.

A Characteristic is a single data value inside a service. Think of a service as a folder and characteristics as files inside it. Each characteristic has:

  • UUID — its unique identifier
  • Value — the actual data (raw bytes)
  • Properties — what operations are allowed on it
READ

You can ask the device for its current value at any time.

WRITE

You can push a new value to the device (to configure or control it).

NOTIFY

The device will push updates to you automatically whenever the value changes.

  1. After selecting a service, click 📃 Discover Characteristics.
  2. Each characteristic is listed with its UUID and property badges.
  3. Click a characteristic to select it — the READ / WRITE / NOTIFY badges at the top of Step 5 will light up according to what it supports.

What happens in the code

// Returns all characteristics within the selected service.
const characteristics = await selectedService.getCharacteristics();

// Each characteristic exposes a .properties object:
// { read: true, write: false, notify: true, ... }
Step 5a — READ

Read a Characteristic Value

Goal: Fetch the current value stored in the characteristic.

A READ operation is a one-shot request: your browser asks the device and the device responds with its current value immediately. The value is returned as raw binary bytes (DataView).

  1. Select a characteristic that has the READ property.
  2. Click 📖 Read Characteristic.
  3. The log will display the value both as:
    • Hex — e.g. 0x48 0x65 0x6C 0x6C 0x6F
    • Text — e.g. Hello (if it's a UTF-8 string)

What happens in the code

// Returns a DataView (raw binary buffer).
const dataView = await characteristic.readValue();

// Convert to hex string for display:
const bytes = new Uint8Array(dataView.buffer);
const hex = Array.from(bytes)
  .map(b => '0x' + b.toString(16).padStart(2,'0').toUpperCase())
  .join(' ');

// Convert to text (UTF-8):
const text = new TextDecoder('utf-8').decode(dataView);
Step 5b — WRITE

Write a Value to a Characteristic

Goal: Send data from the browser to the BLE device.

A WRITE operation sends bytes from the central (browser) to the peripheral. This is how you control or configure a BLE device — for example, turning on an LED, changing a setting, or sending a command.

Text (UTF-8)
hello
Sent as UTF-8 encoded bytes.
Hex bytes
0x01 0xFF 0x2A
Each token is one byte. Prefix with 0x.
  1. Select a characteristic with the WRITE property.
  2. Type a value in the input field (text or hex bytes).
  3. Click ✎ Write.
  4. The log will confirm whether the device acknowledged the write.

What happens in the code

// Convert a plain string to bytes:
const buffer = new TextEncoder().encode('hello');

// Send with acknowledgement (more reliable):
await characteristic.writeValueWithResponse(buffer);

// Or send without acknowledgement (faster, no guarantee):
await characteristic.writeValueWithoutResponse(buffer);
Step 5c — NOTIFY

Subscribe to Notifications

Goal: Receive live data updates pushed by the device automatically.

Instead of repeatedly reading a value (polling), NOTIFY lets the device push updates to you whenever the value changes. This is ideal for continuous sensor data (temperature, heart rate, accelerometer…).

❌ Polling (READ in a loop)

  • Central repeatedly asks for value
  • High power consumption
  • You may miss fast-changing values
  • Wastes bandwidth

✅ NOTIFY (event-driven)

  • Device pushes value when it changes
  • Very low power
  • No missed updates
  • Efficient and real-time
  1. Select a characteristic with the NOTIFY property.
  2. Click 🔔 Subscribe to Notifications.
  3. Every time the device sends a new value, it appears in the log in purple.
  4. Click Unsubscribe when you want to stop receiving updates.

What happens in the code

// Tell the device to start sending notifications.
await characteristic.startNotifications();

// Listen for each incoming value:
characteristic.addEventListener(
  'characteristicvaluechanged',
  (event) => {
    const dataView = event.target.value; // new value as DataView
    console.log(dataView);
  }
);

// Stop notifications when done:
await characteristic.stopNotifications();
ⓘ
Under the hood, subscribing writes a special value to the Client Characteristic Configuration Descriptor (CCCD) on the device, enabling the Notify bit. The device then knows to push updates.
Reference

Key Concepts Summary

Concept Description
BLE Device A physical peripheral that broadcasts BLE advertisements and hosts a GATT server.
GATT Server The database of services and characteristics running on the peripheral.
Service A logical group of related data, identified by a UUID.
Characteristic A single data value within a service. Has a UUID, a value (bytes), and properties.
READ One-shot fetch of a characteristic's current value on demand.
WRITE Send bytes from the central to the peripheral to configure or control it.
NOTIFY Event-driven push from the peripheral whenever the value changes.
UUID Unique identifier. 16-bit for standard services (e.g. 0x180F), 128-bit for custom ones.
DataView JavaScript binary buffer type used to represent raw BLE characteristic values.
CCCD Client Characteristic Configuration Descriptor — enables NOTIFY on the device when written to.
Help

Troubleshooting

The Scan button doesn't open any picker
  • Make sure you are using Chrome or Edge.
  • Make sure Bluetooth is enabled on your computer.
  • On macOS, go to System Settings → Privacy & Security → Bluetooth and allow Chrome/Edge.
  • The page must be served over HTTPS or from localhost / file://.
My device doesn't appear in the list
  • Ensure the device is powered on and in advertising mode (not already connected to another device).
  • Bring the device closer to your computer (stay within ~5 m).
  • Some devices need a button press to start advertising.
Connection fails immediately
  • Try clicking Connect again — BLE connections can fail on the first attempt.
  • Make sure the device is not already connected to another phone or computer.
  • Restart the BLE device and try again.
No services are found
  • Some devices require you to specify service UUIDs in requestDevice() filters instead of using acceptAllDevices: true. Check your device's datasheet.
  • Try disconnecting and reconnecting before discovering services.
Read / Write buttons are greyed out
  • Make sure you have selected a characteristic by clicking on it in the list.
  • A greyed-out button means the selected characteristic does not support that operation — choose a different characteristic.
No notifications are appearing
  • Confirm the characteristic has the NOTIFY badge active.
  • Some devices only send notifications when a physical event occurs (e.g., a button press or sensor change). Trigger the event on the device.
  • Try unsubscribing and subscribing again.
